## Prompt 1：意图识别

### **用途：**
从用户对话中提取关键需求信息

### **完整Prompt：**

```python
INTENT_RECOGNITION_SYSTEM = """你是小遥配配的AI导购助手，专门帮助用户找到合适的电脑配置。

你的任务是从用户的对话中提取以下关键信息：

1. **预算（budget）**：用户的预算范围，如"5000元"、"5000-8001"等
2. **设备类型（device_type）**：desktop（台式机）、laptop（笔记本）、aio（一体机）
3. **用途（usage）**：办公、游戏、设计、编程、学习等
4. **具体需求（requirements）**：对CPU、显卡、内存等的具体要求
5. **品牌偏好（brand）**：联想、华硕、戴尔等
6. **便携性（portable）**：是否需要经常携带（轻薄本/游戏本）

规则：
- 如果用户没有提到某个信息，对应字段返回null
- 尽量从上下文中推断隐含需求
- 预算可以是具体数字或范围
- 设备类型可以从"台式"、"笔记本"等关键词推断
- 用途可以从工作性质推断
- 保持对话友好自然，不要像机器人

请严格按照以下JSON格式返回结果：
```json
{
  "budget": "预算或null",
  "device_type": "设备类型或null",
  "usage": "用途或null",
  "requirements": "具体需求或null",
  "brand": "品牌偏好或null",
  "portable": "便携性需求或null"
}
```"""
```

### **调用方式：**

```python
def extract_intent(
    user_message: str,
    history: Optional[List[Dict[str, str]]] = None,
) -> Dict[str, Any]:
    messages = [
        {"role": "system", "content": INTENT_RECOGNITION_SYSTEM},
    ]

    # 添加历史消息（最近10条）
    if history:
        messages.extend(history[-10:])  # 最近10条历史

    # 添加当前用户消息
    messages.append({"role": "user", "content": user_message})

    result = llm_client.chat_with_structured_output(
        messages=messages,
        output_schema={...},
        temperature=0.3,
    )
    return result
```

### **提取示例：**

| 用户输入 | 提取结果 |
|---------|---------|
| "我想要一台6000左右的游戏笔记本能玩原神" | `{"budget": "6000左右", "device_type": "laptop", "usage": "游戏", "requirements": "玩原神", "brand": null, "portable": null}` |
| "6000左右" | `{"budget": "6000左右", "device_type": null, "usage": null, ...}` |
| "我要联想拯救者" | `{"brand": "联想拯救者", "device_type": "laptop", ...}` |

## Prompt 2：信息完整性检查

### **用途：**
判断当前收集的信息是否足够给出推荐方案

### **完整Prompt：**

```python
IS_INFO_COMPLETE_SYSTEM = """判断当前收集的信息是否足够给出推荐方案。

必须同时满足以下条件才判定信息完整：
1. 有明确的预算范围（budget）
2. 有明确的设备类型（device_type）
3. 有明确的用途（usage）

【重要】严格判断标准：
- 三项信息都必须有，缺一不可
- 不要默认任何值（如不要默认设备类型为"笔记本"）
- 品牌是可选的，不是必需的
- 用户说"没有特别要求"、"都可以"表示对该项无要求，但不算已收集该信息

如果信息足够，返回 {"is_complete": true, "missing_fields": []}
如果信息不足，返回 {"is_complete": false, "missing_fields": ["缺失的字段名"]}

缺失字段：budget（预算）、device_type（设备类型）、usage（用途）

请严格按照JSON格式返回。"""
```

### **调用方式：**

```python
def _check_complete(data: Dict[str, Any]) -> tuple[bool, List[str]]:
    """检查信息完整性（3项必需）"""
    required = ["budget", "device_type", "usage"]
    missing = [f for f in required if not _is_valid(data.get(f))]
    return len(missing) == 0, missing

# 判断值是否有效（非空、非null）
def _is_valid(v: Any) -> bool:
    return v is not None and v != "" and v != "null"
```

### **判断示例：**

| 已收集信息 | 判断结果 |
|-----------|---------|
| `{"budget": "6000左右", "device_type": "laptop", "usage": "游戏"}` | `(True, [])` - 信息完整 |
| `{"budget": "6000左右", "device_type": "laptop"}` | `(False, ["usage"])` - 缺少用途 |
| `{"budget": "6000左右"}` | `(False, ["device_type", "usage"])` - 缺少设备和用途 |

## Prompt 3：对话回复生成

### **用途：**
生成给用户的回复内容和快捷回复选项

### **完整Prompt：**

```python
CHAT_REPLY_SYSTEM = """你是AI导购助手，收集用户电脑需求。{brand_hint}

【已收集信息】
{_format_info(merged)}

【完整性状态】预算:{'✓' if _is_valid(merged.get('budget')) else '✗'} 设备:{'✓' if _is_valid(merged.get('device_type')) else '✗'} 用途:{'✓' if _is_valid(merged.get('usage')) else '✗'}
{f'【缺失项】{missing_hint}' if missing else ''}

用户说：{user_message}

【回复规则】
1. 信息完整时（三项都有✓）→ 复述："好的，{summary_text}。点击下方查看推荐。"
2. 信息不完整时（有✗项）→ 确认新信息，继续询问缺失项，绝对不能说"点击下方查看推荐"

【快捷回复规则】
- 信息完整时，快捷回复固定为：["查看推荐"]，不要添加任何其他选项
- 信息不完整时，快捷回复为缺失信息对应的选项
- 缺用途 → ["办公用途","游戏用途","设计用途","编程用途"]
- 缺设备 → ["台式机","笔记本","一体机"]
- 缺预算 → ["3000元以内","3000-5000元","5000-8000元","8000元以上"]

【重要】信息不完整时，ai_response中绝不能出现"查看推荐"、"为您推荐"等触发推荐的词语。

返回JSON：
```json
{{"ai_response": "回复内容", "quick_replies": ["选项"]}}
```"""
```

**注意**：
- Prompt是**动态构建**的，会根据已收集信息、缺失项、可用品牌等变量实时生成
- `brand_hint`：当用户未指定品牌时，展示店铺可用品牌列表
- `missing_hint`：根据缺失字段生成中文提示，如"请补充：预算、设备类型"
- `summary_text`：复述用户需求，如"预算6000左右、笔记本、用于游戏"

### **调用方式：**

```python
def generate_chat_response(self, user_message: str, extracted: Dict[str, Any],
                           collected_info: Dict[str, Any], brands: Optional[List[str]] = None) -> Dict[str, Any]:
    """
    生成对话回复

    Args:
        user_message: 用户消息
        extracted: 本次提取信息
        collected_info: 累计收集信息
        brands: 可用品牌列表

    Returns:
        {"ai_response": str, "quick_replies": List[str]}
    """
    # 计算状态
    merged = self.merge_info(collected_info, extracted)
    is_complete, missing = self.check_info_complete(merged)

    # 动态构建Prompt（包含品牌提示、缺失提示、已收集信息等）
    # ... 详见后端实现

    response = self.llm.chat([{"role": "system", "content": prompt},
                             {"role": "user", "content": "生成回复"}], temperature=0.7)
    result = _parse_json_response(response)
    return result
```

### **回复示例：**

| 场景 | AI回复 | 快捷选项 |
|-----|--------|---------|
| 用户说"6000左右"（缺设备、用途） | "好的！6000元的预算可以选择台式机或笔记本，请问您主要用来做什么呢？" | ["台式机", "笔记本", "一体机"] |
| 用户说"玩游戏"（缺预算、设备） | "明白了！游戏用途对显卡和CPU要求较高，请问您的预算大概是多少？" | ["3000元以内", "3000-5000元", "5000-8000元", "8000元以上"] |
| 信息完整（三项都有） | "好的，预算6000左右、笔记本、用于游戏。点击下方查看推荐。" | ["查看推荐"] |

**快捷回复规则：**
- 信息完整时：快捷回复固定为 `["查看推荐"]`
- 信息不完整时：根据缺失字段返回特定选项
  - 缺用途 → `["办公用途","游戏用途","设计用途","编程用途"]`
  - 缺设备 → `["台式机","笔记本","一体机"]`
  - 缺预算 → `["3000元以内","3000-5000元","5000-8000元","8000元以上"]`

## Prompt调用流程

### **完整处理流程：**

```python
# app/services/ai/intent_service.py

class IntentRecognitionService:
    def process_message(
        self,
        user_message: str,
        collected_info: Dict[str, Any],
        history: Optional[List] = None,
        available_brands: Optional[List[str]] = None,
    ) -> ChatResponse:
        """处理用户消息的完整流程"""

        # 1. 提取意图
        extracted = self.extract_intent(
            user_message=user_message,
            collected_info=collected_info,
            history=history,
        )

        # 2. 合并信息（新值覆盖旧值）
        merged = self.merge_info(collected_info, extracted)

        # 3. 检查信息完整性（三项必需：budget、device_type、usage）
        is_complete, missing = self.check_info_complete(merged)

        # 4. 生成对话回复
        response = self.generate_chat_response(
            user_message=user_message,
            extracted=extracted,
            collected_info=merged,
            brands=available_brands,
        )

        # 5. 确定下一步动作
        next_action = "recommend" if is_complete else "ask"

        # 6. 处理快捷回复
        if is_complete:
            # 信息完整时，强制只显示"查看推荐"
            quick_replies = ["查看推荐"]
        else:
            # 信息不完整时，移除"查看推荐"（防止AI误生成）
            quick_replies = response.get("quick_replies", [])
            if "查看推荐" in quick_replies:
                quick_replies = [r for r in quick_replies if r != "查看推荐"]

        return ChatResponse(
            ai_response=response.get("ai_response", "好的"),
            extracted_info=ExtractedInfo(**_filter_valid(merged)),
            is_complete=is_complete,
            next_action=next_action,
            quick_replies=quick_replies,
        )
```

### **流程图：**

```
用户输入
    ↓
[1] 意图识别Prompt → 提取需求信息（带历史上下文，最近10条）
    ↓
[2] 合并信息 → 新值覆盖旧值
    ↓
[3] 检查信息完整性 → 判断budget、device_type、usage三项是否齐全
    ↓
[4] 对话回复Prompt → 动态构建，生成回复和快捷选项
    ↓
[5] 确定下一步动作 → ask（继续收集）/ recommend（触发推荐）
    ↓
[6] 处理快捷回复 → 信息完整时强制["查看推荐"]，不完整时移除该选项
    ↓
返回响应
```

**关键点：**
- 信息完整性判断：**必须三项同时满足**（budget、device_type、usage）
- 快捷回复逻辑：信息完整时强制只显示"查看推荐"，防止误触发推荐
- 对话回复Prompt：**动态构建**，包含已收集信息、缺失项、可用品牌等

## 总结

### **设计要点：**

1. **模块化设计**：将功能分离为独立的Prompt，便于维护
2. **结构化输出**：使用`chat_with_structured_output`确保输出格式稳定
3. **温度参数调优**：提取用低温度（0.3），生成用高温度（0.7）
4. **历史记录管理**：保留最近10条对话作为上下文
5. **信息累积机制**：逐步收集用户需求，新值覆盖旧值
6. **动态Prompt构建**：对话回复Prompt根据已收集信息、缺失项、可用品牌等动态生成
7. **快捷回复强制控制**：信息完整时强制只显示"查看推荐"，防止误触发推荐
8. **AI降级策略**：推荐理由优先使用AI生成，失败时降级到模板
