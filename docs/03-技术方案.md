# 技术方案：小遥配配

---

## 1. 技术选型

### 前端

#### **C端（客户端）**
- **框架**：Vue 3.4 + Vite 5
- **UI库**：Ant Design Vue 4.x
- **状态管理**：Pinia
- **路由**：Vue Router 4
- **HTTP客户端**：Axios
- **理由**：
  - Vue 3组合式API开发效率高，代码简洁
  - Vite构建速度快，HMR体验好
  - Ant Design Vue组件丰富，移动端适配良好
  - Pinia轻量级，TypeScript支持完善

#### **B端（商家后台）**
- **框架**：Vue 3.4 + Vite 5
- **UI库**：Ant Design Vue 4.x
- **状态管理**：Pinia
- **路由**：Vue Router 4
- **HTTP客户端**：Axios
- **图表库**：ECharts 5
- **理由**：
  - 与C端技术栈统一，组件和工具函数可复用
  - Ant Design Vue表格、表单功能强大，适合后台管理
  - ECharts图表功能强大，可定制性高

---

### 后端

- **语言**：Python 3.10.15
- **框架**：FastAPI 0.109
- **数据库**：MySQL 8.0
- **ORM**：SQLAlchemy 2.0 + Alembic（数据库迁移）
- **AI框架**：LangChain 0.1
- **LLM**：阿里云通义千问（qwen-plus）
- **缓存**：本地内存缓存（cachetools）
- **文件存储**：阿里云OSS
- **理由**：
  - FastAPI性能优秀，自动生成API文档（Swagger）
  - Python生态丰富，AI库支持完善
  - MySQL稳定可靠，成本低，适合中小型项目
  - SQLAlchemy ORM功能强大，支持复杂查询
  - LangChain提供标准化LLM接口、Prompt模板管理、结构化输出解析，便于模型切换和Prompt优化
  - 本地内存缓存简单高效，无需额外服务

---

### 部署

- **前端（C端+B端）**：阿里云ECS（2核4G，按量付费）
  - 理由：与后端同服务器，内网通信更快，统一管理
- **后端**：阿里云ECS（2核4G，按量付费）
  - 理由：与OSS、通义千问同一云服务商，内网调用更快
- **CDN**：阿里云CDN
  - 理由：加速静态资源访问，降低服务器压力
- **数据库**：阿里云RDS MySQL（基础版，1核1G）
  - 理由：自动备份，高可用，免运维
- **文件存储**：阿里云OSS（标准存储）
  - 理由：价格便宜，CDN加速，免费额度10GB

**架构说明：**

1. **用户访问流程**：
   - 用户访问域名（如 peipei.xiaoyaos.com）
   - 请求先到达阿里云CDN节点
   - CDN缓存静态资源（JS/CSS/图片），直接返回
   - 动态请求回源到ECS服务器

2. **ECS服务器内部**：
   - Nginx作为反向代理，监听80/443端口
   - 静态资源请求 → 直接返回前端构建文件
   - API请求 → 转发到FastAPI后端（端口8001）

3. **域名规划**：
   - `peipei.xiaoyaos.com` → C端前端
   - `peipei-mer.xiaoyaos.com` → B端前端
   - `api.xiaoyaos.com/peipei` → 后端API

4. **内网通信**：
   - ECS ↔ ECS MySQL（本地）
   - ECS ↔ OSS（内网）
   - ECS ↔ 通义千问API（公网，但同区域更快）

## 5. 第三方服务

- [x] **AI框架**：LangChain 0.1
  - 用途：LLM统一接口、Prompt模板管理、结构化输出解析、对话记忆管理
  - 成本：免费（开源框架）

- [x] **AI模型**：阿里云通义千问（qwen-plus）
  - 用途：意图识别、信息提取、推荐理由生成
  - 成本：0.004元/1000 tokens
  
- [x] **文件存储**：阿里云OSS
  - 用途：配置图片、二维码、海报存储
  - 成本：标准存储0.12元/GB/月，免费额度10GB
  
- [x] **数据库**：阿里云RDS MySQL
  - 用途：主数据库
  - 成本：基础版1核1G约60元/月
  
- [x] **本地缓存**：cachetools
  - 用途：对话历史缓存、热门配置缓存、会话管理
  - 成本：免费（使用服务器内存）
  
- [x] **CDN**：阿里云CDN
  - 用途：前端静态资源加速
  - 成本：按流量计费，约0.24元/GB

  - 用途：手机号验证码登录

  - 成本：免费

## 6. 技术风险

| 风险 | 影响 | 应对方案 |
|------|------|---------|
| LangChain版本升级兼容性 | 中 | 1. 锁定版本号<br>2. 关注官方公告<br>3. 保留直接调用API的降级方案 |
| 通义千问API限流 | 高 | 1. 使用本地缓存缓存相同问题的回答<br>2. LangChain快速切换豆包API<br>3. 降级为规则匹配 |
| 通义千问API成本超预算 | 中 | 1. 优化Prompt，减少Token消耗<br>2. 缓存常见问题<br>3. 设置每日调用上限 |
| MySQL成本过高 | 中 | 1. 选择按量付费<br>2. 定期清理历史数据<br>3. 备用自建MySQL |
| OSS存储成本过高 | 低 | 1. 图片压缩（WebP格式）<br>2. 定期清理无用图片<br>3. 免费额度10GB够用 |
| 本地缓存数据丢失 | 中 | 1. 重要数据持久化到数据库<br>2. 缓存失效后从数据库重新加载<br>3. 生产环境可选Redis |
| 并发量过大 | 高 | 1. 使用本地缓存减少数据库压力<br>2. 数据库连接池<br>3. 升级服务器配置 |
| 数据库单点故障 | 高 | 1. 使用RDS自动备份<br>2. 主从复制（P1功能）<br>3. 定期手动备份 |
| AI提取信息不准确 | 中 | 1. 持续优化Prompt<br>2. 人工标注数据微调<br>3. 置信度低时人工介入 |
| 用户隐私泄露 | 高 | 1. 手机号加密存储（AES）<br>2. 接口鉴权<br>3. HTTPS加密传输 |
