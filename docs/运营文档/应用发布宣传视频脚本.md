# 小遥配配 - 产品发布视频脚本

> 参考标准：docs/base/13-演示视频-标准.md
> 预计时长：60-90秒
> 视频风格：技术分享、真实展示、谦逊态度

---

## 视频结构

```
1. 开头钩子（5-10秒）- 问题场景展示
2. 背景故事（10-15秒）- 为什么做这个项目
3. 产品演示（30-40秒）- 核心功能展示
4. 邀请试用（10-15秒）- 开源说明+获取方式
5. 征求意见（5-10秒）- 期待反馈
```

## 核心要点总结

### ✅ 展示，而非空谈
- 真实的产品演示，不是功能罗列
- 画面展示操作过程，而非文字描述

### ✅ 讲故事，而非推销
- 从观察到的真实问题出发
- 解释"为什么做"而非"做了什么"
- 谦逊的技术实践态度

### ✅ 谦逊的态度
- "希望能帮到有需要的同行"
- "有什么可以改进的地方"
- 征求反馈而非硬性推销

### ✅ 低门槛使用
- "永久免费、开源"
- "Docker 一键启动"
- "几分钟就能跑起来"

## 制作建议

### 视频录制
- **工具**：OBS Studio / Camtasia / 系统自带录屏
- **分辨率**：1920x1080（16:9）
- **帧率**：30fps
- **声音**：清晰解说，背景音量适中

### 剪辑要点
- **节奏**：每3-5秒切换一个镜头，保持观看兴趣
- **字幕**：关键信息加中文字幕
- **转场**：简单淡入淡出即可
- **BGM**：轻快、不抢风头

### 发布平台
- **B站**：技术分享向，标题加上"开源"、"AI"
- **YouTube**：英文字幕，国际受众
- **视频号/抖音**：竖屏版本，缩短到30秒
