# 精简MRD：小遥配配 xiaoyaopeipei

## 1. 核心问题（一句话）
**解决什么问题？**
- 电脑店老板重复回答"能玩XX游戏吗""这个配置够不够用"等问题浪费时间且转化率低，需要一个**真AI对话式**的智能导购助手，像真人一样理解客户需求并推荐配置。

## 2. 目标用户（具体画像）

### 主要用户：电脑城档口老板
- **人群特征：**
  - 年龄：30-45岁
  - 经营：电脑城档口/专卖店，1-3人小团队
  - 年收入：20-50万
  - 技术水平：懂硬件配置，不懂编程
  - 痛点认知：知道需要工具提效，但不愿意学复杂系统
  
- **痛点场景：**
  - 周末客流量大，一个人忙不过来，顾此失彼
  - 每天重复回答100+遍同样的问题（"能玩原神吗""这个配置够不够"）
  - 客户不懂配置，用专业术语解释半天还是犹豫不决
  - 新员工不专业，说不清楚配置优势，容易丢单
  - 晚上关门后错过线上咨询商机，第二天客户已经去别家了
  - 朋友圈发配置单没人看，私聊回复太慢客户流失
  
- **当前替代方案：**
  - 口头推荐（效率低，标准不统一，新员工说不清楚）
  - 打印配置单（静态，无法互动，客户看不懂）
  - 微信人工回复（时间成本高，响应慢，晚上无法服务）
  - 通用客服机器人（不懂电脑配置，答非所问，客户体验差）

### 次要用户：装机工作室/淘宝店主
- **人群特征：** 个人创业者，主要靠朋友圈/淘宝接单，时间精力有限
- **痛点场景：** 无法筛选意向客户，客服成本高，转化率低
- **当前替代方案：** 手动回复每个咨询，效率极低，经常半夜还在回消息

## 3. 解决方案（核心功能）

**MVP三个核心功能：**

### 1. **AI对话式需求理解**
- 用户用自然语言描述需求（如："我想要一台5000左右能玩原神的电脑"）
- AI自动提取关键信息：预算、用途、具体需求、设备类型
- 智能追问补充信息，像真人导购一样引导客户
- **技术实现：** 通义千问API进行意图识别和信息提取
- **差异化：** 不是死板的选项问卷，而是自然对话

**示例对话：**
```
用户："我想要一台能玩黑神话悟空的电脑"
AI："黑神话悟空对配置要求比较高，建议至少RTX 4060显卡。
     您的预算大概是多少呢？5000-8000还是8000以上？"
用户："6000左右吧"
AI："好的！除了玩游戏，平时还需要办公或者做设计吗？"
```

### 2. **智能SKU匹配 + 个性化推荐理由**
- **匹配引擎：** 规则匹配（预算筛选 + 标签打分），快速返回Top3方案
- **推荐理由：** AI生成个性化文案，通俗易懂，突出性价比
- 老板可自主维护SKU（价格、库存、标签、推荐方案）
- **技术实现：** 规则引擎（免费快速）+ 大模型生成文案

**示例推荐：**
```
【推荐方案二 - 性价比之选】¥6299 ⭐
• CPU：Intel i5-13400F
• 显卡：RTX 4060 8G
• 内存：16GB DDR4
• 存储：1TB SSD

💡 AI推荐理由：
这个配置非常适合您！RTX 4060显卡可以让原神高画质稳定60帧，
黑神话悟空中画质也能流畅运行。i5-13400F处理器办公多开不卡顿，
16GB内存够用。价格6299元，在您的预算内，性价比最高！
```

### 3. **老板管理后台 + 客户线索管理**
- **SKU管理：** 可视化添加/编辑配置方案，支持快速复制、批量调价
- **问题设置：** 可自定义固定问题（备选模式），问题间无逻辑关系
- **客户线索：** 查看客户需求、预算、选中方案，精准跟进
- **数据看板：** 哪个方案最受欢迎、客户预算分布、转化率统计
- **技术实现：** React Admin + MongoDB

**使用流程：**
```
客户扫码/点链接 
  ↓
AI对话收集需求（2分钟）
  ↓
自动推荐3个方案 + 个性化理由
  ↓
客户选择方案 → 留下联系方式
  ↓
老板后台看到客户需求 → 主动联系成交
```

## 4. 竞品情况

| 竞品 | 优势 | 劣势 | 差异化机会 |
|------|------|------|-----------|
| **京东/天猫配置推荐** | 品牌背书强、流量大、SKU丰富 | 通用化推荐不精准、无法自定义、不适合线下场景、无法沉淀客户 | **专注线下实体店场景**，老板可自定义SKU和价格，适配本地库存，沉淀客户线索 |
| **通用客服机器人**（智齿、网易七鱼） | 功能全面、大厂产品、稳定性好 | 不懂电脑配置、需要大量训练、价格贵（3000元/年起）、配置复杂 | **开箱即用**，无需训练，专注电脑配置场景，价格便宜（99元/月），**真AI对话**而非规则问答 |
| **人工推荐**（传统方式） | 灵活、可议价、能建立信任 | 效率低、标准不统一、无法24小时服务、新员工不专业 | **AI自动化 + 标准化**，提高效率3倍，24小时在线，推荐理由专业且通俗 |
| **Excel配置单/问卷工具** | 简单、免费、易上手 | 静态、无互动、客户看不懂、无法智能匹配 | **AI对话式交互 + 场景化展示**，客户体验好，转化率高，自动生成推荐理由 |

**核心差异化：**
- ✅ **真AI对话**（不是死板的选项问卷，而是自然语言交互）
- ✅ **垂直场景**（只做电脑配置，开箱即用，无需训练）
- ✅ **老板可自主维护SKU**（灵活调价、库存管理、本地化）
- ✅ **个性化推荐理由**（AI生成通俗易懂的文案，提高转化率）
- ✅ **低成本高效率**（99元/月 vs 人工客服3000元/月）

## 6. 风险评估

### 技术风险：⚠️ 低
- **风险点1：** 大模型API不稳定，影响用户体验
  - **应对：** 
    - 接入多个大模型API（通义千问 + 豆包），互为备份
    - 关键对话缓存，减少API调用
    - 降级方案：API故障时切换到固定问答模式
  - **影响：** 低（国产大模型已很稳定，故障率<1%）
  
- **风险点2：** AI推荐不准确，客户不满意
  - **应对：** 
    - MVP阶段规则匹配 + AI生成文案，准确率80%+
    - 收集用户反馈，持续优化prompt
    - 老板可手动调整推荐方案
  - **影响：** 低（规则匹配已能满足大部分场景）

- **风险点3：** 大模型成本超预算
  - **应对：** 
    - 缓存常见问答（命中率60%）
    - 批量调用API（成本降低67%）
    - 使用便宜的模型（豆包0.001元/千tokens）
  - **影响：** 低（成本占比3-5%，可控）

### 市场风险：⚠️⚠️ 中
- **风险点1：** 目标用户（电脑店老板）接受度低，不愿意付费
  - **应对：** 
    - **免费试用1个月** + 地推现场演示 + 成功案例展示
    - 强调"AI"卖点，提升产品档次感
    - 提供7天无理由退款
  - **验证：** 前期找10个种子用户免费试用，收集反馈
  - **影响：** 中（需要教育市场，但痛点真实存在）
  
- **风险点2：** 电脑市场萎缩，需求下降
  - **应对：** 
    - 横向扩展：手机配置推荐、家电推荐、汽车推荐等
    - 纵向深化：增加售后服务、客户管理、进销存等功能
    - 转型B2C：直接服务消费者，绕过商家
  - **影响：** 中（电脑市场虽萎缩但仍有千亿规模，5年内无忧）

- **风险点3：** 大厂入场（京东、天猫推出类似功能）
  - **应对：** 
    - 专注线下场景和小B端，大厂看不上（市场太小）
    - 快速迭代，建立用户壁垒（数据沉淀 + 社区效应）
    - 强调"老板自主维护SKU"的差异化
  - **影响：** 低（大厂不会做这么垂直的小市场，且无法适配线下场景）

### 时间风险：⚠️ 低
- **风险点：** 开发周期过长，错过市场窗口期
  - **应对：** 
    - MVP版本3周完成（SKU管理 + AI对话 + 简单匹配）
    - 快速上线验证，边做边改
    - 优先保证核心功能（AI对话 + 推荐），其他功能后续迭代
  - **影响：** 低（市场需求稳定，无明显窗口期，但要抢在竞品之前）

### 运营风险：⚠️⚠️ 中
- **风险点1：** 地推效率低，获客成本高
  - **应对：** 
    - 多渠道获客：地推 + 线上推广 + 口碑传播
    - 建立用户社群，提高留存和转介绍
    - 提供优质服务，降低流失率
  - **影响：** 中（需要持续投入运营资源）

- **风险点2：** 用户留存率低，续费率不达预期
  - **应对：** 
    - 持续优化产品体验，提高推荐准确率
    - 定期回访用户，收集反馈
    - 提供增值服务（代运营、推广），增加粘性
    - 年付优惠（年付999元，相当于8.3折）
  - **影响：** 中（SaaS产品的核心挑战）

### 合规风险：⚠️ 低
- **风险点：** 大模型生成内容违规（如：虚假宣传、夸大效果）
  - **应对：** 
    - Prompt中明确要求"客观、真实、不夸大"
    - 人工审核高频推荐理由
    - 老板可手动修改AI生成的文案
  - **影响：** 低（电脑配置推荐不涉及敏感内容）

## 7. 成功指标（上线后1个月）

### 核心指标：
- **付费用户数：≥30人**（验证付费意愿，达成盈亏平衡的43%）
- **续费率：≥60%**（验证产品价值，用户愿意持续使用）
- **NPS（净推荐值）：≥40**（验证用户满意度，愿意推荐给朋友）

### 次要指标：
- **注册用户数：≥200人**（验证市场需求）
- **对话完成率：≥70%**（验证AI对话体验，用户不会中途放弃）
- **推荐准确率：≥80%**（用户反馈推荐方案满意，通过问卷调查）
- **客户咨询转化率：≥30%**（完成对话 → 留下联系方式 → 联系老板）
- **平均对话轮次：3-5轮**（验证AI引导能力，太少说明信息不足，太多说明体验差）

### 关键里程碑：
- **第1周：** MVP上线，10个种子用户免费试用，收集初步反馈
- **第2周：** 优化AI对话体验，提高推荐准确率到80%+
- **第3周：** 开始地推（北京中关村/深圳华强北），获取50个注册用户
- **第4周：** 达成30个付费用户，验证商业模式可行

### 失败退出条件：
- **1个月内付费用户 < 10人** → 暂停推广，重新评估产品定位
- **用户反馈问题严重（NPS < 0）** → 产品重构，优化核心体验
- **获客成本 > 200元/人** → 调整推广策略，优化转化漏斗
- **大模型成本 > 收入的20%** → 优化API调用策略，降低成本

- **实时监控：** AI对话成功率、API响应时间、错误率
- **每日统计：** 新增用户、付费转化、对话完成率
- **每周复盘：** 用户反馈、推荐准确率、成本收入比
