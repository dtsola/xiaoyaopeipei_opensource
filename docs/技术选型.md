## 2. 前端技术选型

### 2.1 核心框架
- **Vue 3.4 + Composition API**
  - **选择原因**：现代化响应式框架，开发效率高
  - **核心优势**：组合式API、TypeScript支持、轻量级
  - **替代方案对比**：
    - React：生态丰富但学习曲线较陡
    - Angular：过于庞大，不适合中小型项目

- **Vite 5**
  - **选择原因**：极速的开发服务器和构建工具
  - **核心优势**：HMR快速、打包体积小、开箱即用

### 2.2 UI组件库
- **Ant Design Vue 4.x**
  - **选择原因**：企业级UI组件库，移动端适配良好
  - **核心优势**：组件丰富、设计规范、中文文档完善
  - **替代方案对比**：
    - Element Plus：组件不够丰富
    - Vant：纯移动端，PC端支持弱

### 2.3 状态管理
- **Pinia**
  - **选择原因**：Vue 3官方推荐，API简洁
  - **核心优势**：TypeScript支持、模块化、轻量级
  - **使用场景**：会话信息、对话历史、推荐方案缓存

### 2.4 网络通信
- **Axios**
  - **选择原因**：成熟的HTTP客户端
  - **核心优势**：拦截器、错误处理、自动JSON转换

### 2.5 图表库（B端）
- **ECharts 5**
  - **选择原因**：功能强大的数据可视化库
  - **核心优势**：图表类型丰富、配置灵活、性能优秀
  - **使用场景**：分享统计、设备类型分布、预算分布

## 3. 后端技术选型

### 3.1 Web框架
- **FastAPI 0.109**
  - **选择原因**：现代化Python Web框架，性能优秀
  - **核心优势**：
    - 自动API文档生成（Swagger）
    - Pydantic数据验证
    - 原生异步支持
    - TypeScript友好
  - **替代方案对比**：
    - Django：过于重量级
    - Flask：需要额外插件，开发效率较低

- **Uvicorn + Supervisor**
  - **选择原因**：高性能ASGI服务器 + 进程管理
  - **核心优势**：异步处理、自动重启、日志管理

### 3.2 数据库
- **MySQL 8.0 (本地部署)**
  - **选择原因**：成熟稳定，零额外成本
  - **部署方式**：ECS服务器本地安装
  - **核心优势**：
    - 生态成熟，运维成本低
    - 原生JSON支持
    - 无需额外费用
    - 本地访问速度快
  - **主键策略**：雪花算法（BIGINT 20位）
  - **外键策略**：软外键（应用层控制）
  - **备份策略**：
    - 定时备份脚本（每日凌晨）
    - 备份文件上传到OSS
    - 保留最近7天备份
  - **替代方案对比**：
    - 阿里云RDS：成本高（~60元/月），适合大型项目
    - PostgreSQL：功能更强但运维复杂
    - SQLite：不适合并发场景

### 3.3 ORM框架
- **SQLAlchemy 2.0 + Alembic**
  - **选择原因**：Python最成熟的ORM框架
  - **核心优势**：强大的查询API、数据库迁移、异步支持

### 3.4 缓存方案
- **cachetools (本地内存缓存)**
  - **选择原因**：零成本，性能最高
  - **核心优势**：
    - 无需额外服务器
    - 内存读写，无网络开销
    - 支持TTL、LRU策略
  - **使用场景**：
    - AI对话历史（TTL: 1小时）
    - 热门配置（TTL: 10分钟）
    - 会话信息（TTL: 30分钟）
  - **升级路径**：后期可切换到Redis

### 3.5 日志管理
- **Loguru**
  - **选择原因**：简单易用的日志库
  - **核心优势**：无需配置、日志轮转、异步写入

## 4. AI模型技术选型

### 4.1 智能体框架
- **LangChain 0.1**
  - **选择原因**：标准化LLM接口，便于模型切换和扩展
  - **核心优势**：
    - 统一的LLM调用接口（支持通义千问、豆包等）
    - Prompt模板管理（结构化、版本控制）
    - 结构化输出解析（确保返回JSON格式）
    - 对话记忆管理（简化多轮对话实现）
    - 链式调用（支持多步骤AI处理流程）
  - **使用场景**：
    - 意图识别（Prompt模板）
    - 信息提取（OutputParser）
    - 推荐理由生成（链式调用）
    - 多轮对话（Memory组件）
  - **升级路径**：后期可扩展至LangChain+LangSmith（可观测性）

### 4.2 大语言模型
- **阿里云通义千问 (qwen-plus)**
  - **选择原因**：中文优化，成本低，响应快
  - **技术规格**：
    - 成本：0.004元/1000 tokens
    - 响应速度：国内部署，延迟低
    - 上下文长度：32K tokens
  - **核心优势**：
    - 中文理解能力强
    - 与OSS同一云服务商
    - 稳定性高
  - **替代方案对比**：
    - 文心一言：成本较高（0.012元/1K tokens）
    - ChatGPT：国外部署，延迟高
  - **使用场景**：
    - 意图识别和信息提取
    - 推荐理由生成
    - 方案调整建议

## 6. 部署技术选型

### 6.1 云服务器
- **阿里云ECS (2核4G，按量付费)**
  - **选择原因**：统一部署，管理简单
  - **部署方案**：
    - Nginx：反向代理 + 静态资源服务
    - FastAPI：后端API服务
    - MySQL 8.0：数据库服务（本地部署）
    - 前端：静态文件部署
  - **核心优势**：
    - 所有服务统一管理
    - 本地通信，速度快
    - 成本低，无额外数据库费用

### 6.2 数据库部署
- **MySQL 8.0 (ECS本地部署)**
  - **安装方式**：apt/yum包管理器安装
  - **配置优化**：
    - InnoDB缓冲池：1GB
    - 最大连接数：200
    - 慢查询日志：开启
  - **安全配置**：
    - 仅允许本地访问（127.0.0.1）
    - 强密码策略
    - 定期安全更新
  - **监控方案**：
    - 磁盘空间监控
    - 连接数监控
    - 慢查询分析

### 6.3 CDN加速
- **阿里云CDN**
  - **选择原因**：加速静态资源访问
  - **核心优势**：全球节点、HTTPS支持、按流量计费

## 10. 技术选型总结

### 核心原则
1. **成本优先**：选择性价比高的方案，MySQL本地部署节省成本
2. **快速开发**：选择团队熟悉的技术
3. **性能足够**：满足100+ QPS需求
4. **易于维护**：生态成熟、文档完善
5. **可扩展性**：预留升级路径

### 技术亮点
✅ **Vue 3 + FastAPI**：前后端分离，开发效率高
✅ **LangChain + 通义千问**：标准化AI接口，便于模型切换
✅ **雪花算法**：分布式ID生成，支持扩展
✅ **软外键**：高性能，易于分库分表
✅ **本地缓存**：零成本，性能优秀
✅ **MySQL本地部署**：零额外成本，访问速度快
✅ **自动化文档**：FastAPI自动生成API文档

### 升级路径

| 阶段 | 并发量 | 升级方案 |
|------|--------|---------|
| **初期** | < 100 QPS | 单服务器 + 本地MySQL + 本地缓存 |
| **成长期** | 100-500 QPS | Redis缓存 + MySQL主从复制 |
| **扩展期** | 500-1000 QPS | 阿里云RDS + 负载均衡 + 分库分表 |

### MySQL运维建议

**日常维护：**
- 定时备份脚本（每日凌晨3点）
- 备份文件上传到OSS
- 监控磁盘空间和连接数
- 定期优化慢查询

**升级时机：**
- 数据量超过10GB
- 并发连接数超过100
- 需要高可用保障
- 运维成本超过RDS费用
