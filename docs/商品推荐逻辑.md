# 商品推荐逻辑文档

> 本文档描述小遥配配后端SKU推荐系统的完整实现逻辑

---

## 一、推荐流程概述

```
用户对话 → AI意图识别提取需求 → 信息完整性检查 → SKU匹配评分 → 生成推荐理由 → 返回推荐列表
```

## 五、6个字段匹配规则

代码位置: [match_service.py:365-377](../xiaoyaopeipei-user-merchant-backend/app/services/match_service.py#L365-L377)

### 5.1 字段处理方式

| 字段 | 处理方式 | 说明 |
|------|---------|------|
| **budget** 预算 | 评分40% | 软约束，预算范围匹配，超/低按差距扣分 |
| **device_type** 设备类型 | **严格筛选** | 只推荐同类型SKU，不同类型不参与评分 |
| **usage** 用途 | 评分权重30% | 影响配置分（CPU/GPU/内存的权重分配） |
| **requirements** 具体需求 | **传给AI** | 不参与匹配，传给推荐理由服务让AI体现 |
| **brand** 品牌 | 评分10% | 偏好匹配100分，无偏好80分，不匹配60分 |
| **portable** 便携性 | 额外加分0-10分 | 轻薄需求按重量加分，续航需求有电池加分 |

### 5.2 设备类型严格筛选规则

```python
# 严格过滤：如果用户指定了设备类型，只匹配同类型的SKU
if user_device_type:
    skus = [sku for sku in skus if sku.device_type.lower() == user_device_type.lower()]
```

**原因**：设备类型是硬性约束，台式机用户不应看到笔记本推荐。

### 5.3 便携性加分规则

代码位置: [match_service.py:296-343](../xiaoyaopeipei-user-merchant-backend/app/services/match_service.py#L296-L343)

| 用户需求 | SKU特征 | 加分 |
|---------|---------|------|
| 轻薄/便携 | 重量<1.5kg | +10分 |
| 轻薄/便携 | 重量<2.0kg | +7分 |
| 轻薄/便携 | 重量<2.5kg | +4分 |
| 轻薄/便携 | 重量<3.0kg | +2分 |
| 轻薄/便携 | 台式机/一体机 | 0分（直接返回） |
| 续航需求 | 有电池信息 | +5分 |
| 续航需求 | 台式机 | -5分 |

**加分上限**：10分

### 5.4 requirements 传给AI处理

代码位置: [recommend_service.py:76-93](../xiaoyaopeipei-user-merchant-backend/app/services/ai/recommend_service.py#L76-L93)

用户说的具体需求（如"要16G内存"、"需要独立显卡"）不参与匹配计算，而是传递给AI推荐理由生成服务：

```python
def _format_user_needs(self, needs: Dict[str, Any]) -> str:
    parts = []
    if needs.get("requirements"):
        parts.append(f"具体要求：{needs['requirements']}")
    if needs.get("portable"):
        parts.append(f"便携性：{needs['portable']}")
    return "、".join(parts)
```

AI会在推荐理由中自然体现这些要求。
