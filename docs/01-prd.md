# PRD：小遥配配 MVP版本

## 1. 产品定位（一句话）
为 **电脑店老板** 提供 **AI对话式客户需求收集和智能配置推荐（支持台式机/笔记本/一体机）** 的 **SaaS智能导购助手**

---

## 2. 用户故事（核心场景）

### **场景1：客户自助咨询（C端用户视角）**
- **作为** 想买电脑但不懂配置的客户
- **我想要** 通过简单对话就能找到适合我的电脑（台式机/笔记本/一体机）
- **以便于** 不用等老板有空，也不用听一堆专业术语，2分钟就能看到推荐方案

### **场景2：老板被动获客（B端老板视角）**
- **作为** 电脑店老板
- **我想要** 客户扫码后AI自动收集需求并推荐配置（台式机/笔记本/一体机）
- **以便于** 我不用重复回答同样的问题，晚上关门后也能获取客户线索，提高转化率

### **场景3：老板管理多类型配置方案**
- **作为** 电脑店老板
- **我想要** 快速录入和管理台式机、笔记本、一体机的配置方案（价格、库存、标签）
- **以便于** 系统能根据我的实际库存和设备类型自动推荐，避免推荐了没货或类型不匹配的配置

### **场景4：客户灵活表达需求**
- **作为** 想买电脑的客户
- **我想要** 用自己的话表达需求（如："我想要一台6000左右的游戏笔记本能玩原神"）
- **以便于** 不用回答一堆固定问题，AI能理解我的需求并快速推荐

### **场景5：老板分享推广**
- **作为** 电脑店老板
- **我想要** 生成专属二维码和链接，方便分享给客户
- **以便于** 我能把二维码贴在店里，或者发朋友圈推广，让更多客户扫码咨询

## 3. 功能清单（MVP范围）

### **P0功能（必须有）- 3周内完成**

#### **C端（客户端）：**
- [x] **功能1：AI对话式需求收集（智能动态对话）**
  - 用户扫码/点链接进入对话页面
  - AI开场白引导用户表达需求
  - **支持五花八门的输入方式：**
    * 一次性说完所有需求："我想要一台6000左右的游戏笔记本能玩原神"
    * 只说预算："6000左右"
    * 直接说品牌："我要联想拯救者"
    * 反问AI："你们有什么推荐吗？"
    * 模糊需求："我要一台好用的电脑"
  - **AI意图识别（通义千问API）：**
    * 自动提取：预算、设备类型、用途、具体需求、品牌偏好、便携性等
    * 判断信息完整度（缺什么问什么）
    * 智能追问补充信息（1-2个问题）
  - **支持用户随时修改需求：**
    * 用户查看方案后说"有没有台式机？"
    * AI识别修改意图，重新匹配方案
  - 对话历史记录（用户可以返回修改）
  - 快捷回复按钮（如："台式机🖥️" "笔记本💻" "一体机🖥️"）

- [x] **功能2：智能配置推荐（Top3方案，按设备类型匹配）**
  - 根据用户需求和设备类型，自动匹配Top3配置方案
  - 每个方案展示：配置详情、价格、图片、设备类型标识
  - **台式机**显示：CPU、显卡、内存、存储、主板、电源、机箱
  - **笔记本**显示：品牌、型号、CPU、显卡、内存、存储、屏幕、重量、续航
  - **一体机**显示：品牌、型号、CPU、显卡、内存、存储、屏幕尺寸
  - AI生成个性化推荐理由（通俗易懂，突出性价比）
  - 方案对比功能（参数对比表）
  - 用户可选择心仪方案

- [x] **功能3：留下联系方式**
  - 用户选择方案后，引导留下手机号/微信
  - 表单验证（手机号格式校验）
  - 提交成功提示："老板会尽快联系您"

#### **B端（商家后台）：**
- [x] **功能4：SKU配置管理（支持三种设备类型，动态表单）**
  - 添加配置时选择设备类型：台式机/笔记本/一体机
  - **台式机表单**：CPU、显卡、内存、存储、主板、电源、机箱、价格、库存
  - **笔记本表单**：品牌、型号、CPU、显卡、内存、存储、屏幕、重量、续航、价格、库存
  - **一体机表单**：品牌、型号、CPU、显卡、内存、存储、屏幕尺寸、价格、库存
  - 配置标签设置（gaming/office/design/budget/portable/brand/desktop/laptop/aio）
  - 配置图片上传
  - 库存状态管理（在售/缺货）
  - 快速编辑/删除配置
  - 批量复制功能（快速创建相似配置）

- [x] **功能5：客户线索管理**
  - 查看客户列表（时间倒序）
  - 客户详情：需求、预算、设备类型、选中方案、联系方式
  - 完整对话记录展示
  - 线索状态标记（待跟进/已联系/已成交/已放弃）
  - 备注功能（记录沟通内容）
  - 一键拨打电话/复制微信号

- [x] **功能6：账号注册/登录**
  - 用户名 + 密码注册
  - 用户名 + 密码登录
  - 账号基本信息设置（店铺名称、联系方式）
  - 密码加密存储（bcrypt）
  - 登录状态保持（JWT Token）

- [x] **功能7：分享功能**
  - 老板生成专属二维码（可下载，可打印贴在店里）
  - 老板生成专属链接（可复制，可发朋友圈）
  - 分享海报生成（带二维码，3种模板可选）

### **技术流程（AI对话 + 匹配）：**
```
用户输入 → 通义千问API（意图识别）
    ↓
提取关键信息：
  {
    "budget": "6000左右",
    "device_type": "笔记本",
    "usage": "游戏",
    "requirements": "玩原神",
    "brand": null,
    "portable": null
  }
    ↓
判断信息完整度：
  - 必填：budget, device_type, usage ✅
  - 可选：requirements, brand, portable
  - 决策：可以推荐，但追问1个细节问题
    ↓
AI生成追问："您需要经常携带吗？还是主要在家用？"
    ↓
用户回答 → 更新需求画像
    ↓
规则匹配引擎：
  - 设备类型筛选（laptop）
  - 预算筛选（5400-6600，±10%）
  - 标签打分（gaming: +50, portable: +20）
  - 库存过滤（只推荐有货的）
  - 品牌匹配（如果用户提到品牌）
  - 排序（按得分从高到低）
    ↓
返回Top3方案（同一设备类型）
    ↓
通义千问API（批量生成推荐理由）
    ↓
返回给用户展示
```

## 7. 非功能需求

### **性能要求：**
- 首屏加载时间 < 2秒（C端对话页面）
- AI对话响应时间 < 3秒（通义千问API调用 + 意图识别）
- 推荐方案生成时间 < 5秒（规则匹配 + AI生成推荐理由）
- 后台管理页面加载 < 3秒
- 二维码生成时间 < 2秒
- 海报生成时间 < 5秒
- 支持并发：100个用户同时对话（MVP阶段）

### **兼容性要求：**
- **浏览器兼容：**
  - Chrome 90+
  - Safari 14+
  - 微信浏览器（iOS/Android）
  - 手机QQ浏览器
- **设备兼容：**
  - iOS 12+
  - Android 8.0+
  - 屏幕尺寸：375px - 414px（移动端优先）
- **后台管理：**
  - PC端浏览器（1280px+）
  - 支持Chrome、Edge、Safari

### **安全要求：**
- HTTPS加密（全站SSL证书）
- 数据加密：
  - 用户密码哈希加密（bcrypt，加盐）
  - 用户手机号加密存储
  - 微信号加密存储
- 接口安全：
  - JWT Token认证（有效期7天）
  - 接口限流（防刷）：每个IP每分钟最多10次请求
  - 登录失败次数限制（5次锁定30分钟）
  - AI接口调用限流：每个用户每分钟最多5次
- 隐私保护：
  - 用户隐私协议
  - 数据不对外泄露
  - 符合《个人信息保护法》
  - 对话记录加密存储

### **可用性要求：**
- 系统可用性 > 99%（月度）
- 数据备份：每日自动备份
- 故障恢复时间 < 1小时
- AI接口故障时，降级为规则匹配

### **可维护性要求：**
- 代码规范：ESLint + Prettier
- 接口文档：Swagger自动生成
- 日志系统：记录关键操作和错误
- 监控告警：服务器异常自动通知
- Prompt版本管理：每次修改Prompt记录版本

### **扩展性要求：**
- 数据库设计支持横向扩展
- API设计RESTful，易于扩展
- 大模型API可切换（通义千问 → 豆包 → 其他）
- Prompt模板化，易于调整和优化
- 支持未来增加新设备类型（如：平板电脑、服务器）
