## 一、统计实现方式

### 1.1 实现模式

**触发事件后增量统计**（Incremental Event-Triggered Statistics）

系统采用**事件驱动**的增量统计方式，当关键业务事件发生时，实时更新对应的统计计数器。

### 1.2 设计优势

| 优势 | 说明 |
|------|------|
| **实时性** | 事件发生后立即更新数据库，无需等待定时任务 |
| **轻量** | 不需要定时任务轮询，降低服务器负载 |
| **精确** | 每次事件都准确记录，不会丢失数据 |
| **简单** | 逻辑清晰，易于维护和调试 |

## 二、数据存储结构

### 2.1 数据库表设计

表名：`share_stats`（分享统计表）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | BIGINT | 主键（雪花算法） |
| `merchant_id` | BIGINT | 商家ID |
| `date` | DATE | 统计日期 |
| `scan_count` | INT | 扫码次数 |
| `consult_count` | INT | 咨询次数 |
| `lead_count` | INT | 线索数量 |
| `device_type_stats` | JSON | 设备类型统计 |
| `budget_stats` | JSON | 预算区间统计 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间 |

### 2.2 唯一性约束

```sql
UNIQUE KEY `unique_merchant_date` (`merchant_id`, `date`)
```

**说明**: 每个商家每天只有一条统计记录

## 三、事件触发流程

### 3.1 事件类型与对应方法

| 业务事件 | 触发时机 | CRUD方法 | 更新字段 |
|----------|----------|----------|----------|
| **扫码事件** | 用户扫描商家二维码 | `increment_scan_count()` | `scan_count += 1` |
| **咨询事件** | 用户开始AI对话 | `increment_consult_count()` | `consult_count += 1` + 更新设备类型/预算 |
| **留资事件** | 用户提交联系方式 | `increment_lead_count()` | `lead_count += 1` |

### 3.2 流程图

```
用户行为 → 业务接口 → 调用CRUD增量方法 → 更新统计记录 → 返回结果
   ↓                                      ↓
扫码二维码                        获取/创建今日记录
开始对话                          实时更新计数器
提交留资                          提交事务
```

def increment_scan_count(
    self, db: Session, merchant_id: int

#### 咨询统计 (Lines 116-166)

```python
def increment_consult_count(
    self,
    db: Session,
    merchant_id: int,
    device_type: Optional[str] = None,
    budget: Optional[str] = None,
) -> ShareStat:
    """增加咨询数（并记录设备类型和预算分布）"""
    stat = self.get_today_stat(db, merchant_id=merchant_id)
    stat.consult_count += 1

    # 更新设备类型统计
    if device_type:
        device_stats = json.loads(stat.device_type_stats or '{}')
        device_stats[device_type] = device_stats.get(device_type, 0) + 1
        stat.device_type_stats = json.dumps(device_stats, ensure_ascii=False)

    # 更新预算分布统计
    if budget:
        budget_stats = json.loads(stat.budget_stats or '{}')
        budget_key = self._get_budget_key(budget)
        budget_stats[budget_key] = budget_stats.get(budget_key, 0) + 1
        stat.budget_stats = json.dumps(budget_stats, ensure_ascii=False)

    db.commit()
    db.refresh(stat)
    return stat
```

## 四、趋势计算逻辑

### 4.1 计算公式

```python
趋势值 = ((今日数值 - 昨日数值) / 昨日数值) × 100
```

### 4.2 计算时机

仅在 **period="today"** 时计算趋势数据

### 4.3 边界处理

| 场景 | 处理方式 |
|------|----------|
| 昨日有数据，今日有数据 | 正常计算趋势百分比 |
| 昨日 = 0，今日 > 0 | 返回 `100.0`（增长100%） |
| 昨日 = 0，今日 = 0 | 返回 `0`（无变化） |
| 昨日 > 0，今日 = 0 | 返回负数（下降100%） |

### 4.4 趋势字段

| 字段名 | 说明 | 计算基础 |
|--------|------|----------|
| `consult_trend` | 咨询数趋势 | (今日consult_count - 昨日consult_count) / 昨日consult_count × 100 |
| `lead_trend` | 线索数趋势 | (今日lead_count - 昨日lead_count) / 昨日lead_count × 100 |
| `conversion_trend` | 转化率趋势 | (今日转化率 - 昨日转化率) / 昨日转化率 × 100 |

### 4.5 代码实现 (Lines 305-344)

```python
# ========== 计算趋势（较昨日） ==========
consult_trend = 0
lead_trend = 0
conversion_trend = 0

if period == "today":
    yesterday_stats = self.get_by_merchant_and_date(db, merchant_id, yesterday)

    if yesterday_stats:
        # 计算咨询数趋势
        if yesterday_stats.consult_count > 0:
            consult_trend = round(
                ((total_consult - yesterday_stats.consult_count) / yesterday_stats.consult_count) * 100, 2
            )
        elif total_consult > 0:
            consult_trend = 100.0  # 昨日0，今日有数据

        # 计算线索数趋势
        if yesterday_stats.lead_count > 0:
            lead_trend = round(
                ((total_lead - yesterday_stats.lead_count) / yesterday_stats.lead_count) * 100, 2
            )
        elif total_lead > 0:
            lead_trend = 100.0

        # 计算转化率趋势
        yesterday_conversion_rate = 0.0
        if yesterday_stats.consult_count > 0:
            yesterday_conversion_rate = round(
                (yesterday_stats.lead_count / yesterday_stats.consult_count) * 100, 2
            ) / 100

        if yesterday_conversion_rate > 0:
            conversion_trend = round(
                ((conversion_rate - yesterday_conversion_rate) / yesterday_conversion_rate) * 100, 2
            )
        elif conversion_rate > 0:
            conversion_trend = 100.0
```

## 七、统计周期支持

### 7.1 支持的周期

| period值 | 说明 | 日期范围 |
|----------|------|----------|
| `today` | 今日 | 当天 |
| `week` | 本周 | 最近7天 |
| `month` | 本月 | 最近30天 |
| `all` | 全部 | 全部时间 |

### 7.2 趋势计算说明

**仅 `period="today"` 时返回趋势数据**，因为趋势是相对于昨日的对比，其他周期没有对比基准。

## 八、注意事项

### 8.1 数据一致性

- 使用数据库事务确保原子性
- 每次 `increment` 操作后立即 `commit()`
- 使用 `refresh()` 获取最新数据

### 8.2 性能优化

- `merchant_id + date` 唯一索引，查询高效
- JSON字段用于存储动态统计（设备类型、预算分布）
- 避免了定时任务扫描全表

### 8.3 扩展性

如需新增统计维度，只需：
1. 在 `share_stats` 表添加新字段
2. 在对应的 `increment_*` 方法中更新该字段
3. 在 `get_summary_stats()` 中汇总该字段
